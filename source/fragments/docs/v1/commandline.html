Command Line

Help is available to you in your terminal application:

  • trio -h: display general help information
  • trio -h <command>: display command specific help information

The following commands are available to you in your terminal application:

  • trio new <path/to/project>: creates an empty project in path/to/project
  • trio build | b: builds your site targeting the public folder
  • trio build | b -i: incrementally builds your site targeting the public folder
  • trio build | b -w: builds your site targeting the public folder any time a change is made to a file in the root/source folder
  • trio build | b -iw: incrementally builds your site targeting the public folder any time a change is made to a file in the root/source folder
  • trio build | b -ws: builds your site targeting the public folder any time a change is made to a file in the root/source folder and serves it in the default browser
  • trio build | b -iws: incrementally builds your site targeting the public folder any time a change is made to a file in the root/source folder and serves it in the default browser
  • trio release | r: builds your site for release targeting the release folder
  • trio release | r -b: builds your site for release targeting the release folder with cache busting applied
  • trio serve | s: serves your site from the public folder in the default browser
  • trio serve | s -r: serves your site from the release folder in the default browser

Build Vs. Release

Running build targets the root/public folder, and is what you do while you are developing your site. Trio generate a CSS map file and generate pages whose fragments were marked as a WIP.

Running release targets the root/release folder, and is what you do prior to deploying your site to its server. Trio doesn't generate a CSS map file, doesn't generate pages whose fragments were marked as a WIP, cleans your site's pages of all their data-trio-* tag attributes and provides the option to cache bust the generated site.
