Configuration

You may have noticed while reading through these documentation pages that hardly any mention at all has been made of configuration. That's because Trio was designed with the lofty goal of minimizing the amount of configuration that it requires to a bare minimum. Who likes configuration anyway, right?

At this time there are only 2 required configuration properties, and even they are given logical default values, so you can literally install Trio, create a new project, and you are good to go.

The content of the root/trio.json configuration file is made available to your tag-based callback assets in the metadata site.userConfig property, including any custom properties that you may have added.

Predefined Configuration Properties

  • baseURL: type string, default "" - see Base URL in Page Generation for more information
  • blogFolderName: type string, default "blog" - see For Blog Article Pages in Page Generation for more information
