Metadata

When building your site, Trio creates an organized collection of metadata objects that it creates from your project's assets. Trio makes this metadata available to you through the various arguments that it passes to your project's tag-based callbacks.

After each build of your project, Trio persists most of the metadata that it has collected to a file named root/trio.manifest.json. You can view the contents of this file to assist you when coding tag-based callbacks.

Below is an inventory of the metadata that is available to you.

Please note that beginning with v1.0.0-rc.5, Trio no longer generates a unique id property for fragments due to the expense in terms of the development time required to maintain their integrity during incremental builds. In its place, you should now use the fragment's url property, which is guaranteed to always be intrinsically unique per fragment.

fragment

An object that contains all the metadata associated with a fragment.

You can access the current fragment in your tag-based callbacks using the callback's asset parameter. You can access all non article fragments in your tag-based callbacks using the callback's site parameter's frags property. You can access all article fragments in your tag-based callbacks using the callback's site parameter's articlesCatalog property.

fragment.path

The path to the fragment file in the source/fragment folder. It is used internally by Trio during page generation.

fragment.mtimeMs

The timestamp that represent the time, in milliseconds, when the fragment was last modified. It is used internally by Trio to determine the files to be included when building incrementally. See Incremental Build for more information.

fragment.type

A string that indicates the type of the fragment. It is used internally by Trio for determining the paths of dependencies between project assets when building incrementally.

fragment.isStale

A boolean that indicates if the fragment was marked as stale. It is used internally by Trio when building incrementally.

fragment.matter

The fragment's front matter. See Front Matter for more information.

fragment.destPath

Please note that, beginning with v1.0.0-rc.5, destPath is now generated in all lowercase.

The path to the page in the the public folder.

fragment.url

The URL of the page.

fragment.collection

Only relevant for fragments that are part of a collection. See Collections for more information. Introduced in v2.0.0.

fragment.articleDate

Only relevant for articles, is a string which contains the date of the article in "YYYY-MM-DD" format.

fragment.nextArticleUrl

Only relevant for articles, is the URL to the next article.

fragment.previousArticleUrl

Only relevant for articles, is the URL to the previous article.

fragment.relatedArticlesByCategory

Only relevant for articles that have categories, is an object that catalogs other articles related to this article by its category.

fragment.relatedArticlesByCategory.category

The article's category.

fragment.relatedArticlesByCategory.related

An array of articles related to this article in descending alphabetical order by article date and article title.

fragment.relatedArticlesByCategory.related[n].articleDate

The date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByCategory.related[n].url

The article's URL.

fragment.relatedArticlesByCategory.related[n].title

The article's title.

fragment.relatedArticlesByCategory.related[n].excerpt

The article's front matter's excerpt.

fragment.relatedArticlesByTag

Only relevant for articles that have tags. Is an array that catalogs other articles related to this article by its tags. Catalog items are in ascending alphabetical order by tag.

fragment.relatedArticlesByTag[n].tag

The tag.

fragment.relatedArticlesByTag[n].related

An array of articles related to this article by fragment.relatedArticlesByTag[n].tag in descending alphabetical order by article date and article title.

fragment.relatedArticlesByTag[n].related[n].articleDate

The date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByTag[n].related[n].url

The article's URL.

fragment.relatedArticlesByTag[n].related[n].title

The article's title.

fragment.relatedArticlesByTag[n].related[n].excerpt

The article's excerpt.

fragment.relatedArticlesByTagFlattened

An array that catalogs all other articles related to this article and is similar to fragment.relatedArticlesByTag but flattened for when you aren't concerned about each related article's tag.

fragment.relatedArticlesByTagFlattened[n].articleDate

The date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByTagFlattened[n].url

The article's URL.

fragment.relatedArticlesByTagFlattened[n].title

The article's title.

fragment.relatedArticlesByTagFlattened[n].excerpt

The article's excerpt.

timestamp

The date and time of the current build in "YYYY-MM-DD HH:MM:SS" format.

version

The version of Trio used to build the project in semantic versioning format.

buildType

Either "development" or "release".

userConfig

A copy of the project's trio.json file.

frags

An array of all non article fragments. See Property fragment above for information pertaining to each array item's format.

wipsCount

An integer, the site's total number of "work in progress" fragments.

wips

An array of all the "work in progress" fragments. See Property fragment above for information pertaining to each array item's format.

articlesCount

An integer, the total number of blog articles.

articlesCatalog

Only relevant for projects that have articles. Is an array that catalogs the site's blog article's in ascending alphabetical order by article date and article title. See Property fragment above for information pertaining to each array item's format.

categoriesCatalog

Only relevant for projects that have articles that have categories. Is an array that catalogs the site's blog articles in ascending alphabetical order by category.

categoriesCatalog[n].category

The article's category.

categoriesCatalog[n].related

An array of related articles in descending alphabetical order by article date and article title.

categoriesCatalog[n].related[n].articleDate

The date of the article in "YYYY-MM-DD" format.

categoriesCatalog[n].related[n].url

The article's URL.

categoriesCatalog.related[n].title

The article's title.

categoriesCatalog.related[n].excerpt

The article's excerpt.

tagsCatalog

Only relevant for projects that have articles that have tags. Is an array that catalogs the site's blog articles in ascending alphabetical order by tag.

tagsCatalog[n].tag

The tag.

tagsCatalog[n].related

An array of related articles in descending alphabetical order by article date and article title.

tagsCatalog[n].related[n].articleDate

The date of the article in "YYYY-MM-DD" format.

tagsCatalog[n].related[n].url

The article's URL.

tagsCatalog.related[n].title

The article's title.

tagsCatalog.related[n].excerpt

The article's excerpt.

dataCatalog

A hash that catalogs the site's JSON data files in root/source/data along with their content. Each property in the hash represents a single JSON data file. You can access any JSON data file in your tag-based callbacks using the callback's site parameter's dataCatalog property. See JSON Data for more information.
