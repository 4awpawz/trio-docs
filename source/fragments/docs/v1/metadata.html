Metadata

When building your site, Trio gathers metadata from your project's assets and massages it into organized collections of data that you can use in your tag-based callbacks.

After each build of your project, Trio persists most of the metadata that it has collected to a file named root/trio.manifest.json. You can view the contents of this file to assist you when coding tag-based callbacks.

Below is a full inventory of all the metadata that is available to you.

fragment

fragment is an object containing all the metadata associated with a fragment.

fragment.path

The path property is the path to the fragment file in the source/fragment folder. It is used internally by Trio during page generation.

fragment.mtimeMs

The mtimeMs property is a timestamp that represent the time, in milliseconds, when the fragment was last modified. It is used internally by Trio to determine a file's staleness for incremental builds.

fragment.type

The type property is a string which indicates the type of fragment that a file is and is used internally by trio for incremental builds.

fragment.isStale

The isStale property is a boolean which indicates if the fragment was marked as stale and is used internally by trio for incremental builds.

fragment.matter

The matter property is the fragment's front matter.

fragment.matter.content

The matter.content property is a string which contains the fragment's HTML content.

fragment.matter.data

The matter.data property is an object with properties for each property declared in the front matter.

fragment.matter.isEmpty

The matter.isEmpty property is a boolean which indicates if the front matter's content property is empty.

fragment.matter.excerpt

The matter.excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

fragment.id

The id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from frags, articlesCatalog, categoriesCatalog, tagsCatalog, fragment.relatedArticlesByCategory, fragment.relatedArticlesByTag and fragmentRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

fragment.destPath

The destPath property contains the path to the page in the the public folder.

fragment.url

The url property contains the URL of the page.

fragment.articleDate

Only relevant for blog sites, the articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

fragment.nextArticleUrl

Only relevant for blog sites, the nextArticleUrl property contains the URL to the next article.

fragment.previousArticleUrl

Only relevant for blog sites, the previousArticleUrl property contains the URL to the previous article.

fragment.relatedArticlesByCategory

Only relevant for blog sites that use categories, the relatedArticlesByCategory property is an object that catalogs other articles related to this article by its category.

fragment.relatedArticlesByCategory.category

The relatedArticlesByCategory.category property is the page's category.

fragment.relatedArticlesByCategory.related

The relatedArticlesByCategory.related property is an array of related articles in descending order by article date and article title.

fragment.relatedArticlesByCategory.related[n].articleDate

The related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByCategory.related[n].url

The related[n].url property is the related article's URL.

fragment.relatedArticlesByCategory.related[n].title

The related[n].title property is the related article's title.

fragment.relatedArticlesByCategory.related[n].id

The related[n].id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from frags, articlesCatalog, categoriesCatalog, tagsCatalog, fragment.relatedArticlesByCategory, fragment.relatedArticlesByTag and fragmentRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

fragment.relatedArticlesByCategory.related[n].excerpt

The related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

fragment.relatedArticlesByTag

Only relevant for blog sites that use tags, the relatedArticlesByTag property is an array that catalogs other articles related to this article grouped in ascending order by tag.

fragment.relatedArticlesByTag[n].tag

The relatedArticlesByTag[n].tag property is an article tag.

fragment.relatedArticlesByTag[n].related

The relatedArticlesByTag[n].related property is an array of articles related to this article by fragment.relatedArticlesByTag[n].tag.

fragment.relatedArticlesByTag[n].related[n].articleDate

The related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByTag[n].related[n].url

The related[n].url property is the related article's URL.

fragment.relatedArticlesByTag[n].related[n].title

The related[n].title property is the related article's title.

fragment.relatedArticlesByTag[n].related[n].id

The related[n].id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from frags, articlesCatalog, categoriesCatalog, tagsCatalog, fragment.relatedArticlesByCategory, fragment.relatedArticlesByTag and fragmentRelatedArticlesByTagFlattened. fragment.relatedArticlesByTag[n].related[n].excerpt

The related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

fragment.relatedArticlesByTagFlattened

The relatedArticlesByTagFlattened property is an array that catalogs other articles related to this article and is similar to fragment.relatedArticlesByTag but flattened for when you aren't concerned about each related article's tag.

fragment.relatedArticlesByTagFlattened[n].related

The relatedArticlesByTagFlattened[n].related property is an array of relate articles.

fragment.relatedArticlesByTagFlattened[n].related[n].articleDate

The related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

fragment.relatedArticlesByTagFlattened[n].related[n].url

The related[n].url property is the related article's URL.

fragment.relatedArticlesByTagFlattened[n].related[n].title

The related[n].title property is the related article's title.

fragment.relatedArticlesByTagFlattened[n].related[n].id

The related[n].id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from frags, articlesCatalog, categoriesCatalog, tagsCatalog, fragment.relatedArticlesByCategory, fragment.relatedArticlesByTag and fragmentRelatedArticlesByTagFlattened. fragment.relatedArticlesByTagFlattened.related[n].excerpt

The related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

timestamp

The timestamp property is the date and time of the current build in "YYYY-MM-DD HH:MM:SS" format.

version

The version property is the version of Trio used to build the project in semantic versioning format.

buildType

The buildType property can be either "development" or "release", depending on the build options.

userConfig

The userConfig property contains the contents of the trio.json file.

frags

The frags property is an array of all non blog-article fragments. See Property fragment above for information pertaining to each array item's format.

wipsCount

The wipsCount property is an integer whose value is the site's total number of "work in progress" fragments.

wips

The wips property is an array of all the "work in progress" fragments.

articlesCount

The articlesCount property is an integer whose value is the site's total number of blog articles.

articlesCatalog

Only relevant for blog sites, the articlesCatalog property is an array that catalogs the site's blog article's sorted in ascending order by article date and article title. See Property fragment above for information pertaining to each array item's format.

categoriesCatalog

Only relevant for blog sites that use categories, the categoriesCatalog property is an array that catalogs the site's blog articles grouped in ascending order by category.

categoriesCatalog[n].category

The categoriesCatalog[n].category property is the category.

categoriesCatalog[n].related

The categoriesCatalog[n].related property is an array of related articles in descending order by article date and article title.

categoriesCatalog[n].related[n].articleDate

The related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

categoriesCatalog[n].related[n].url

The related[n].url property is the related article's URL.

categoriesCatalog.related[n].title

The related[n].title property is the related article's title.

categoriesCatalog.related[n].id

The related[n].id property is an integer whose value is the unique ID. It is used internally by Trio. It can be used to filter for fragments from frags, articlesCatalog, categoriesCatalog, tagsCatalog, fragment.relatedArticlesByCategory, fragment.relatedArticlesByTag and fragmentRelatedArticlesByTagFlattened. Do not rely on this value to remain constant from one build to next. Always retrieve it from the front matter directly.

categoriesCatalog.related[n].excerpt

The related[n].excerpt property is a string which contains the HTML, if any, from the front matter's excerpt.

tagsCatalog

Only relevant for blog sites that use tags, the tagsCatalog property is an array that catalogs the site's blog articles grouped in ascending order by tag.

tagsCatalog[n].tag

The tagsCatalog[n].tag property is the tag.

tagsCatalog[n].related

The tagsCatalog[n].related property is an array of related articles in descending order by article date and article title.

tagsCatalog[n].related[n].articleDate

The related[n].articleDate property is a string which contains the date of the article in "YYYY-MM-DD" format.

dataCatalog

The dataCatalog property is an array that catalogs the site's JSON files in root/source/data along with their content. Each item in the array represents a single JSON file. Please note that this catalog is not persisted to the root/trio.manifest.json file. See JSON Data for more information.
