Configuration

You may have noticed while reading through these documentation pages that hardly any mention at all has been made of configuration. That's because Trio was designed with the lofty goal of minimizing the amount of configuration that it requires to a bare minimum. Who likes configuration anyway, right?

At this time there are no required configuration properties, so you can literally install Trio, create a new project, and you are good to go.

The content of the root/trio.json configuration file, including any custom properties that you may have declared, can be accessed in tag-based callbacks using the metadata site.userConfig property.

Predefined Configuration Properties

baseURL

  • optional, type string, default "" - see Base URL in Linking Conventions for more information
