Configuration

There are no required configuration properties!

You can add your own custom configuration properties to the root/trio.json configuration file and Trio will expose them to your tag-based callbacks via the site.userConfig metadata property.

Predefined Optional Configuration Properties

baseURL

  • optional, type string, default "" - see Base URL in Linking Conventions for more information

permalinks

  • optional, type array, default [] - see Defining Permalinks In trio.json in Permalinks for more information

sassSourceMaps

  • optional, type object, default { "development": true, "release": true } - see SCSS/SASS/CSS for more information

ignore

  • optional, type string | array, default [] - use string to ignore a single folder, an array of strings to ignore more than one folder - see Project Structure for more information
